# AI Agent Guidelines

## Commit Message Conventions

When generating commit messages, AI agents should use **present tense** rather than past tense.

### Examples

✅ Correct (Present Tense):
- "Add new feature for user authentication"
- "Fix bug in payment processing"
- "Update documentation for API endpoints"

❌ Incorrect (Past Tense):
- "Added new feature for user authentication"
- "Fixed bug in payment processing"
- "Updated documentation for API endpoints"

### Reasoning

Using present tense in commit messages follows the Git convention established by Linus Torvalds and is consistent with how most developers write commit messages. This makes the AI-generated commits blend naturally with human-generated ones in the project history.
